    Características y ventajas de Apache

    • Servidor web HTTP de código abierto
    • Se desarrolla dentro del proyecto HTTP Server (httpd) de la Apache Software Foundation
    • Implementa el protocolo HTTP/1.1
    • Modular
    • Multi-plataforma (Windows, Linux, MacOS)
    • Muy estable
    • Tradicionalmente solución más popular (soporte)

    Gestión del Servicio Apache

    • service apache2 [start|stop|restart|reload|status]
    • start: iniciar el servicio apache2
    • stop: parar el servicio apache2
    • restart: reiniciar el servicio apache2
    • reload: reinicia el servicio apache2, sin matar las conexiones activas
    • status: presentar el estado del servicio apache2

    Estructura del Servidor Web Apache

    • Contenido aplicaciones Web del servidor web Apache /var/www
    • Ruta virtual host / site por defecto /var/www/html
    • Configuración servidor web Apache /etc/apache2
    • Módulos servidor Web
    • Ficheros configuración (Apache, Virtual Host)
    • Registro ficheros de actividad (Servidor, Virtual Host) logs /var/log/apache2

    Estructura / Tipo de Directorios

    • Directorios de CONTENIDO:
      • /var/www: ruta por defecto para alojamiento Web (htdocs)
      • /var/www/midominio: ruta para los host virtuales
      • /var/www/html: ruta del virtual host / site por defecto de Apache
    • Directorio de REGISTROS:
      • /var/log/apache2/access.log: log que registra todas las peticiones hechas al servidor Web
      • /var/log/apache2/error.log: log que registra todos los errores producidos en el servidor Web
    • Directorios de CONFIGURACIÓN:
      • /etc/apache2: directorio por defecto de los archivos de configuración (Apache/conf/)
      • /etc/apache2/apache2.conf: configuración principal global de Apache (Apache/conf/httpd.conf)
      • /etc/apache2/ports.conf: archivos los puertos por los que Apache escucha, por defecto, 80 / 443 si SSL está habilitado
      • /etc/apache2/sites-available: archivos de configuración de los VH. Apache no usará los archivos de configuración que se encuentren en este directorio si no están vinculados al directorio «sites-enabled»
      • /etc/apache2/sites-enabled/: VH del servidor web
      • /etc/apache2/conf-available/ y /etc/apache2/conf-enabled/: archivos de configuración que no pertenecen a ningún VH
      • /etc/apache2/mods-available/ y /etc/apache2/mods-enabled/: contienen los módulos habilitados (enabled) y los disponibles (available) para su uso
      • Archivos *.load: Archivos carga específicos de algún módulo en particular
      • Archivos *.conf: Archivos de configuración de estos módulos
