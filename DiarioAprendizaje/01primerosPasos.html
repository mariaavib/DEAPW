    • Primeros pasos

    • git clone
      • git clone [remote_repo] [directory_local] ➙ se usa para copiar un repositorio remoto en local.
      • git clone --branch [tag] [remote_repo] ➙ se usa para copiar únicamente la versión de la tag indicada
      • git clone --branch --single-branch [branch_name] [remote_repo] ➙ se usa para copiar únicamente la rama remota que se indica.
    • git pull [servidor_remoto] fusiona los cambios del repositorio remoto con el directorio de trabajo local. Ejecuta fetch para bajar los cambios y luego merge para fusionarlos automáticamente.
      • git pull [servidor_remoto] [rama] ➙ se baja una rama del servidor remoto
      • git pull --verbose [servidor_remoto] ➙ muestra un proceso detallado
    • git fetch [nombre_servidor_remoto] recupera datos del proyecto remoto que no tienes en tu repositorio local. Actualiza las referencias a todas las ramas remotas sin modificar tu trabajo actual ni unir automáticamente los cambios.
      • git chekout fetch_HEAD ➙ recupera datos del proyecto remoto que no tienes en tu repositorio local. Actualiza las referencias a todas las ramas remotas sin modificar tu trabajo actual ni unir automáticamente los cambios.
      • git checkout –b [new_Branch] ➙ recupera datos del proyecto remoto que no tienes en tu repositorio local. Actualiza las referencias a todas las ramas remotas sin modificar tu trabajo actual ni unir automáticamente los cambios.
    • git fetch
      • git checkout –b [nombre_nueva_branch] ➙ salto a la rama que tienen los cambios del git fetch
      • git checkout [ID_commit] ➙ si se quiere generar una nueva rama con la foto bajada del remoto.

    Comandos adicionales para consolidar los cambios:

    • git diff [branch_origen] [branch_destino]
    • git pull [rama]
    • git merge [rama]
