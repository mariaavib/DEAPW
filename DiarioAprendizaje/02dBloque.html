    • Directivas de Bloque en Apache

    Las directivas de bloque permiten configurar opciones específicas para directorios, ficheros y ubicaciones específicas dentro del servidor Apache.

    Bloque <Directory>

    Las directivas dentro de este bloque aplican únicamente al directorio especificado, a sus subdirectorios y contenidos.

    <Directory /ruta_absoluta>
    // Aquí van las opciones
    </Directory>

    Ejemplo de Opciones:

    • Options: Controla las características del servidor.
    • AllowOverride: Permite o no sobrescribir directivas mediante archivos .htaccess.
    • Order: Define el orden de evaluación de las reglas allow y deny.
    • Allow/Deny: Permite o deniega accesos.
    • Require: Controla el acceso basado en usuarios, IPs, etc.

    Bloque <Files>

    Se aplica exclusivamente a los archivos indicados.

    <Files "evg.html">
    // Aquí van las opciones
    </Files>

    Se puede combinar con expresiones regulares:

    <FilesMatch "^\.ht">
    // Aquí van las opciones
    </FilesMatch>

    Bloque <Location>

    Controla opciones basadas en URLs solicitadas, independientemente del path físico en el servidor.

    <Location /ruta>
    // Aquí van las opciones
    </Location>

    • Procesa las directivas después de <Directory> y puede sobrescribirlas.
    • Usado comúnmente con SetHandler para definir manejadores.

    Ejemplo de SetHandler:

    • default-handler: Maneja contenido estático.
    • cgi-script: Ejecuta scripts CGI.
    • server-info: Muestra información de configuración del servidor.
    • server-status: Muestra el estado del servidor.

    Orden de Lectura en Tiempo de Ejecución

    El orden en que Apache procesa las directivas es el siguiente:

    • <Directory> se lee primero.
    • Archivos .htaccess se procesan junto a <Directory> y pueden sobrescribirlas si está permitido.
    • <Files> se procesa después.
    • <Location> es procesado al final.
