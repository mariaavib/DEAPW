    • Iniciar el proceso

      GIT STATUS

    • git status➙ muestra la lista de los archivos que se han cambiado junto con los archivos que están por ser preparados o confirmados.

      • GIT ADD

    • git add [archivo/patrón] ➙ añade el archivo a staging

      • GIT COMMIT

    • git commit -m 'mesaje' ➙guarda los archivos al repositorio local
    • git commit --amend -m 'Nuevo mensaje' ➙agrega cambios al último commit

      • GIT DIFF

    • git diff ➙ mostrar diferencias entre los archivos
    • git diff <commit><commit> ➙ mostrar diferencias entre los commits mostrar diferencias entre los commits

      • GIT REMOTE

    • git remote ➙ mostrará una lista con los nombres de los remotos que hayas especificado (verás al menos ‘origin’)
    • git remote add [nombre_repositorio][URL] ➙
    • git remote show [nombre_repositorio] ➙ ver más información acerca de un repositorio remoto en particular.
    • git remote rename[old][new] ➙ renombrar una referencia a un repositorio remoto
    • git remote rm[nombre_repositorio] ➙ eliminar alguna referencia del repositorio local y no elimina su historial del sistemas de versiones.
      • git rm --cached ➙ mueve los archivos que indiquemos al estado untracked
      • git rm --force ➙ elimina los archivos de git y del disco

      GIT RESET

    • git reset HEAD [fichero]➙ quita los archivos de staged y los devuelve a sus estado anterior (unstaged).

      • GIT BRANCH

    • git branch -M main ➙ la rama master local pasa a llamarse main, como la de por defecto de GITHub.

      • GIT PUSH

    • git push -u [rama_remota][rama_local] ➙ genera un canal de comunicacion de subida (upstream), entre la rama local y la remota:
    • git push -u origin main
    • git push -u [nombre_servidor_remoto][nombre_rama] ➙ comando que te permite enviar tu proyecto al servidor remoto para compartir desde la nube.

      • GIT CLEAN

    • git clean ➙hace una eliminación permanente del sistema de archivos (recursos untracked), similar a ejecutar la utilidad rm de línea de comandos.
      • git clean -n ➙ indica los ficheros que eliminaría
      • git clean -f ➙ elimina los ficheros

      GIT CHECKPUT

    • git checkput [id_commit] ➙ Cambia el directorio de trabajo al estado exacto del id_commit. Permite consultar, editar archivos, compilar y realizar pruebas sin afectar el estado actual del proyecto. Para volver al desarrollo normal, usa git checkout main.

      • GIT RESET

    • git reset HEAD [file] ➙ devuelve el fichero al estado untracked.
    • git reset [--soft || --mixed || --hard][HEAD^ || id_commit] ➙ elimina los commits que se le indica por argumento, y en función del modo.
    • git revert [HEAD^ || id_commit] ➙ deshace cambios en el historial de confirmaciones creando una nueva "confirmación de reversión". Esto mantiene la integridad del historial y evita errores de sincronización con el servidor remoto.
      • -e || --edit te permite incluir un mensaje en el commit, es la acción por defecto.
      • --no-edit añade el mensaje por defecto
      • revierte lo indicado pero no realiza el nuevo commit

      GIT REVERT

    • git revert [nombre_rama]~5 [nombre_rama]~3 ➙ revierte los cambios de los commit 5 y 3 commit hacia atrás
    • git revert [nombre_rama]~5..[nombre_rama]~3 ➙ revierte los cambios desde el 5 commit hacia atrás (incluido), hasta el 3 commit hacia atrás
    • git revert HEAD~4 ➙revierte los cambios del 3o commit último, sin contar donde está el puntero HEAD.

      • GIT TAG

    • git tag ➙ muestra las etiquetas que existen y se han definido en el repositorio local git.
    • git tag -a [nombre_etiqueta] -m [“Mensaje”] ➙ muestra cómo se define la etiqueta a la que hace referencia, indicando a qué commit corresponde su versión.
    • git tag -a [id_commit] -m [“Mensaje”] ➙ genera la etiqueta con el nombre dado en el commit que se indica por su id.
